# Paradox Registry (Informational/Causal Stress Tests)

Purpose: convert high-level "paradoxes" into explicit, testable stress cases for the framework.

A paradox is useful only if it can be formalized as a contradiction risk with measurable pass/fail conditions.

---

## Registry format

Each paradox entry must include:

- `case_id`
- `name`
- `paradox_type` (causal / informational / relativistic / quantum / computational)
- `setup` (minimal formal scenario)
- `apparent_contradiction`
- `framework_resolution` (how contradiction dissolves under constraints)
- `marker_stress_set`
- `falsifier` (what result would break the framework)
- `prediction_hook` (what novel prediction this case enables)
- `status` (open / active / resolved / falsified)

---

## Active cases

### PRX-001 — Bootstrap information illusion (self-originating message)

- **Type:** causal + informational
- **Setup:** a message appears to encode knowledge not generated by any forward process in the modeled system.
- **Apparent contradiction:** information with no causal provenance.
- **Framework resolution:** require complete provenance graph; apparent loop must reduce to hidden initial/boundary conditions + inference.
- **Marker stress:** C1, I1, O2, A2
- **Falsifier:** reproducible protocol where provenance cannot be assigned under any admissible forward model.
- **Prediction hook:** measurable provenance entropy reduction from expanded observability.
- **Status:** open

### PRX-002 — Delayed-choice inference vs retrocausality confusion

- **Type:** quantum + causal
- **Setup:** measurement choices made later appear to redefine earlier state descriptions.
- **Apparent contradiction:** future measurement causes past physical state.
- **Framework resolution:** update epistemic state, not ontic past; enforce no-signaling.
- **Marker stress:** C1, P2, O3
- **Falsifier:** operational signaling channel from future choice to past control variable.
- **Prediction hook:** quantified divergence between epistemic-update metrics and ontic-causation metrics.
- **Status:** open

### PRX-003 — Relativistic ordering disagreement

- **Type:** relativistic + informational
- **Setup:** two observers in distinct frames disagree on event ordering for spacelike-separated events.
- **Apparent contradiction:** incompatible causal narratives.
- **Framework resolution:** require frame-explicit statements and invariant content transfer bounds.
- **Marker stress:** P1, C1, I1
- **Falsifier:** frame-invariant claim that implies superluminal transfer.
- **Prediction hook:** frame-dependent decoding performance envelopes with frame-invariant admissibility.
- **Status:** open

### PRX-004 — Maxwell-demon-style informational free lunch

- **Type:** informational + thermodynamic
- **Setup:** algorithm claims persistent entropy reduction/work extraction without accounting measurement + memory reset costs.
- **Apparent contradiction:** net computational gain at no energetic cost.
- **Framework resolution:** include full measurement/control/erasure accounting.
- **Marker stress:** P3, I2, C2
- **Falsifier:** repeatable net gain beyond accounting bounds under agreed assumptions.
- **Prediction hook:** explicit efficiency ceilings for observer-control loops.
- **Status:** open

### PRX-005 — Oracle leakage masquerading as prediction

- **Type:** computational + empirical
- **Setup:** model appears highly predictive due to hidden future data leakage.
- **Apparent contradiction:** unreal forecasting accuracy inconsistent with causal information budget.
- **Framework resolution:** enforce strict input cutoff, pre-registration, and audit trail.
- **Marker stress:** E1, E2, E4, C1, A2
- **Falsifier:** high lift persists after leakage controls and independent replication.
- **Prediction hook:** calibrated degradation profile when leakage paths are removed.
- **Status:** active

## Case lifecycle policy

1. `open`: defined but not yet tested
2. `active`: linked to ongoing experiment/prediction
3. `resolved`: contradiction dissolved with evidence
4. `falsified`: framework broken for this case

A `falsified` case must trigger immediate matrix update and governance review.

## Required links
